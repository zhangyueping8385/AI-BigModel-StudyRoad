## 📚 一、代码概述

本代码实现了**余弦相似度（Cosine Similarity）**算法，用于计算两个向量之间的相似程度。这是自然语言处理、推荐系统、信息检索等领域最常用的相似度计算方法之一。

### 1.1 核心功能

- ✅ 实现向量点积计算
- ✅ 实现向量模长计算
- ✅ 实现余弦相似度计算
- ✅ 提供多组测试用例

### 1.2 余弦相似度的用途

| 应用领域 | 具体用途 |
|---------|---------|
| **NLP** | 文本相似度、语义匹配 |
| **推荐系统** | 基于用户/物品的协同过滤 |
| **信息检索** | 查询与文档的相关度排序 |
| **计算机视觉** | 图像相似度、特征匹配 |
| **数据分析** | 聚类、分类、异常检测 |

## 🔧 二、核心代码解析

### 2.1 点积计算

```python
"""
计算两个向量的点积
两个向量对应位置相乘之和
"""
def get_dot(vec_a,vec_b):
    if len(vec_a) != len(vec_b):
        print("两个向量纬度不一样！")
    sum_dot = 0
    for a,b in zip(vec_a,vec_b):
        sum_dot += (a * b)
    return sum_dot
```

**点积（Dot Product）的定义**：

两个向量对应位置元素相乘，然后求和。

```
向量 a = [a₁, a₂, a₃, ..., aₙ]
向量 b = [b₁, b₂, b₃, ..., bₙ]

点积 = a₁×b₁ + a₂×b₂ + a₃×b₃ + ... + aₙ×bₙ
```

**几何意义**：
- 点积 = |a| × |b| × cos(θ)
- 反映了两个向量在方向上的"投影"程度

**示例**：
```
vec_a = [1, 2, 3]
vec_b = [4, 5, 6]
点积 = 1×4 + 2×5 + 3×6 = 4 + 10 + 18 = 32
```

### 2.2 模长计算

```python
"""
计算向量的模
某向量每个数字平方开根号
"""
def get_norm(vec_a):
    sum_norm = 0
    for i in vec_a:
        sum_norm += (i * i)
    return np.sqrt(sum_norm)
```

**模长（Norm / Magnitude）的定义**：

向量各元素平方和的平方根，也就是向量的"长度"。

```
向量 a = [a₁, a₂, a₃, ..., aₙ]

模长 = √(a₁² + a₂² + a₃² + ... + aₙ²)
```

**几何意义**：
- 向量在 n 维空间中的长度
- 也叫 L2 范数（欧几里得范数）

**示例**：
```
vec_a = [3, 4]
模长 = √(3² + 4²) = √(9 + 16) = √25 = 5
```

### 2.3 余弦相似度计算

```python
"""
计算余弦相似度
两个向量的点积 / 两个向量的模长的乘积
"""
def cosine_similarity(vec_a,vec_b):
    return get_dot(vec_a, vec_b) / (get_norm(vec_a) * get_norm(vec_b))
```

**余弦相似度公式**：

```
            a · b
cos(θ) = -----------
          |a| × |b|
```

其中：
- `a · b` 是点积
- `|a|` 和 `|b|` 是模长
- `cos(θ)` 就是余弦相似度

### 2.4 测试用例

```python
if __name__ == '__main__':
    vec_a = [0.5, 0.5]
    vec_b = [0.7, 0.7]
    vec_c = [0.7, 0.5]
    vec_d = [-0.6, -0.5]
    
    print(cosine_similarity(vec_a, vec_b))  # 方向相同，相似度=1
    print(cosine_similarity(vec_a, vec_c))  # 方向接近，相似度高
    print(cosine_similarity(vec_a, vec_d))  # 方向相反，相似度=-1
```

## 💡 三、关键知识点

### 3.1 余弦相似度的取值范围

余弦相似度的取值范围是 **[-1, 1]**：

| 取值 | 含义 | 说明 |
|-----|------|------|
| **1** | 完全相同 | 两个向量方向完全一致 |
| **0** | 正交（无关） | 两个向量垂直，没有相似性 |
| **-1** | 完全相反 | 两个向量方向完全相反 |

**示意图（二维空间）**：

```
      相似度=1        相似度>0        相似度=0        相似度<0
        ↑               ↗               ↑               ↖
        |              /                |                \
        |             /                 |                 \
    ────+────     ────+────     ────+────     ────+────
        |             \                 |                 /
        |              \                |                /
        |               ↘               ↓               ↙
```

### 3.2 余弦相似度 vs 欧氏距离

| 对比项 | 余弦相似度 | 欧氏距离 |
|-------|-----------|---------|
| **衡量的是** | 方向的相似性 | 距离的远近 |
| **取值范围** | [-1, 1] | [0, +∞) |
| **对长度敏感** | 不敏感 | 敏感 |
| **归一化** | 自动归一化 | 需要手动归一化 |
| **适用场景** | 文本相似度、方向比较 | 空间距离、聚类 |

**为什么文本相似度用余弦？**

因为我们关心的是"主题是否相似"，而不是"词频是否一样多"。两篇文章即使长度差异很大，只要主题相同，余弦相似度就高。

### 3.3 余弦相似度的几何解释

余弦相似度衡量的是**两个向量夹角的余弦值**：

- 夹角越小 → 余弦值越接近 1 → 越相似
- 夹角 90 度 → 余弦值为 0 → 不相关
- 夹角越大 → 余弦值越接近 -1 → 越相反

> 💡 想象一下：两个向量就像两支箭，箭头指向的方向越接近，它们就越"相似"。余弦相似度就是在衡量这个"方向的接近程度"。

## 🎯 四、文本相似度应用

### 4.1 词袋模型 + 余弦相似度

最简单的文本相似度计算方法：

1. **分词**：把文本拆成词语
2. **构建词表**：收集所有出现的词
3. **向量化**：统计每个词出现的次数，形成向量
4. **计算余弦**：用余弦相似度比较两个向量

**示例**：

```
文本A: "我 爱 编程"
文本B: "我 爱 学习"

词表: [我, 爱, 编程, 学习]

向量A: [1, 1, 1, 0]
向量B: [1, 1, 0, 1]

余弦相似度 = (1×1 + 1×1 + 1×0 + 0×1) / (√3 × √3) = 2/3 ≈ 0.67
```

### 4.2 TF-IDF + 余弦相似度

进阶版本，给重要的词更高的权重：

- **TF（词频）**：词在文档中出现的频率
- **IDF（逆文档频率）**：词在所有文档中的稀有程度
- **TF-IDF**：TF × IDF，词的重要性得分

这样，"的"、"是"这类常用词权重低，"人工智能"、"机器学习"这类专业词权重高。

## ⚠️ 五、注意事项

1. **向量维度必须相同**：两个向量的长度要一样，否则无法计算
2. **零向量问题**：如果向量全是 0，模长为 0，会出现除零错误，需要特殊处理
3. **只看方向不看长度**：余弦相似度不关心向量的"大小"，只关心"方向"。这在某些场景下是优点，某些场景下是缺点
4. **稀疏向量效率**：如果向量很稀疏（大部分是0），可以用稀疏矩阵优化计算
5. **高维空间**：在高维空间中，所有向量的余弦相似度都趋近于 0（维度灾难）

## 📝 六、NumPy 简洁实现

实际项目中通常直接用 NumPy 计算，更简洁高效：

```python
import numpy as np

def cosine_similarity_np(vec_a, vec_b):
    """使用 NumPy 计算余弦相似度"""
    dot_product = np.dot(vec_a, vec_b)      # 点积
    norm_a = np.linalg.norm(vec_a)          # 模长
    norm_b = np.linalg.norm(vec_b)          # 模长
    return dot_product / (norm_a * norm_b)
```

**一行代码版**：
```python
from sklearn.metrics.pairwise import cosine_similarity
similarity = cosine_similarity([vec_a], [vec_b])[0][0]
```

## 🔍 七、扩展知识

### 7.1 其他相似度计算方法

| 方法 | 公式/原理 | 适用场景 |
|-----|----------|---------|
| **余弦相似度** | cos(θ) | 文本、向量方向比较 |
| **欧氏距离** | √(Σ(aᵢ-bᵢ)²) | 空间距离、聚类 |
| **曼哈顿距离** | Σ|aᵢ-bᵢ| | 城市距离、L1正则 |
| **杰卡德相似度** | 交集/并集 | 集合相似度、推荐系统 |
| **皮尔逊相关系数** | 协方差/(σ₁×σ₂) | 相关性分析、推荐系统 |

### 7.2 余弦距离

有时候会用"余弦距离"而不是相似度：

```
余弦距离 = 1 - 余弦相似度
```

取值范围是 [0, 2]：
- 完全相同 → 距离 = 0
- 完全相反 → 距离 = 2

这样更符合"距离"的直觉：越近越相似。

## 💡 八、直观理解

记住这几个例子就能理解余弦相似度：

| 向量A | 向量B | 余弦相似度 | 说明 |
|------|------|-----------|------|
| [1, 0] | [1, 0] | 1 | 完全相同 |
| [1, 0] | [0, 1] | 0 | 垂直，不相关 |
| [1, 0] | [-1, 0] | -1 | 完全相反 |
| [1, 1] | [2, 2] | 1 | 方向相同，长度不同，相似度还是1 |

> 💡 最后一点很重要：余弦相似度不关心向量的"长度"，只关心"方向"。所以 [1,1] 和 [100,100] 的余弦相似度是 1，因为它们方向完全一致。
