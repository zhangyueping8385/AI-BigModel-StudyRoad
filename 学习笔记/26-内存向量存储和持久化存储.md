## 📚 一、代码概述

本代码演示了**LangChain 中两种向量存储方式**的使用：
- **InMemoryVectorStore**：内存向量存储，速度快但程序退出数据丢失
- **Chroma**：持久化向量存储，数据保存到磁盘，下次可继续使用

同时演示了向量的**增、删、查**基本操作。

### 1.1 核心功能

- ✅ InMemoryVectorStore 内存向量存储
- ✅ Chroma 持久化向量存储
- ✅ 添加文档向量（add_documents）
- ✅ 删除向量（delete）
- ✅ 相似度搜索（similarity_search）

## 🔧 二、核心代码解析

### 2.1 导入相关类

```python
from langchain_chroma import Chroma
from langchain_core.vectorstores import InMemoryVectorStore
from langchain_community.embeddings import DashScopeEmbeddings
from langchain_community.document_loaders import CSVLoader
```

| 导入类 | 说明 |
|-------|------|
| `Chroma` | Chroma 向量数据库（持久化） |
| `InMemoryVectorStore` | 内存向量存储（临时） |
| `DashScopeEmbeddings` | 通义千问的 Embedding 模型 |
| `CSVLoader` | CSV 文件加载器 |

### 2.2 内存向量存储（InMemoryVectorStore）

```python
vector_store = InMemoryVectorStore(
    embedding=DashScopeEmbeddings()
)
```

**特点**：
- 数据存在内存中，速度极快
- 程序退出后数据丢失
- 适合开发测试、临时使用
- 不需要安装额外数据库

**参数**：
- `embedding`：Embedding 模型，用于把文本转成向量

### 2.3 持久化向量存储（Chroma）

```python
vector_store = Chroma(
    collection_name="test",              # 集合名称，类似数据库名
    embedding_function=DashScopeEmbeddings(),  # 向量化函数
    persist_directory='./data/chroma_db'  # 存储路径
)
```

**特点**：
- 数据保存到磁盘，持久化存储
- 程序退出后数据还在，下次可以加载继续用
- 适合生产环境、长期使用
- 需要安装 `langchain-chroma` 和 `chromadb` 库

**参数说明**：

| 参数 | 类型 | 说明 |
|-----|------|------|
| `collection_name` | str | 集合名称，类似数据库的表名 |
| `embedding_function` | Embeddings | 向量化模型函数 |
| `persist_directory` | str | 数据存储的目录路径 |

> 💡 注意：Chroma 的参数名是 `embedding_function`，而 InMemoryVectorStore 是 `embedding`，名字不一样，别搞混了。

### 2.4 加载文档数据

```python
loader = CSVLoader(
    file_path="data/info.csv",
    encoding="utf-8",
    source_column="source"
)
documents = loader.load()
```

用 CSVLoader 加载数据，准备存入向量库。

### 2.5 添加向量（新增）

```python
vector_store.add_documents(
    documents=documents,
    ids=["id"+str(i) for i in range(1, len(documents)+1)]
)
```

**参数**：
- `documents`：要添加的 Document 对象列表
- `ids`：（可选）为每个文档指定唯一 ID，方便后续删除或更新

**ID 生成方式**：
```python
ids=["id1", "id2", "id3", ...]
```

> 💡 如果不指定 ids，系统会自动生成随机 ID。但手动指定 ID 更方便后续管理。

### 2.6 删除向量

```python
vector_store.delete(["id1", "id2"])
```

传入要删除的文档 ID 列表，即可删除对应的向量。

### 2.7 相似度搜索（查询）

```python
result = vector_store.similarity_search(
    "Python是不是很简单呀",
    2  # 返回结果的个数
)
print(result)
```

**参数**：
- 第一个参数：查询的问题/文本
- 第二个参数（k）：返回最相似的前 k 个结果

**返回值**：
- `List[Document]`：Document 对象的列表，按相似度从高到低排序

## 💡 三、关键知识点

### 3.1 什么是向量存储？

**向量存储（Vector Store）** 是专门用来存储和检索向量的数据库。

**工作原理**：

```
文档 → Embedding 模型 → 向量（一串数字）→ 存入向量库
问题 → Embedding 模型 → 向量（一串数字）→ 相似度搜索 → 返回最相似的文档
```

**为什么需要向量存储？**

| 问题 | 传统方式 | 向量存储 |
|-----|---------|---------|
| 语义搜索 | 只能关键词匹配 | 能理解语义，找意思相近的 |
| 搜索速度 | 数据量大了慢 | 专门优化，速度快 |
| 相似度计算 | 自己算，麻烦 | 内置支持，直接用 |

### 3.2 内存存储 vs 持久化存储

| 对比项 | InMemoryVectorStore | Chroma |
|-------|---------------------|--------|
| **存储位置** | 内存 | 磁盘 |
| **持久化** | ❌ 程序退出就没了 | ✅ 永久保存 |
| **速度** | ⚡ 极快 | 🚀 快 |
| **适用场景** | 开发测试、临时数据 | 生产环境、长期使用 |
| **安装依赖** | 不需要 | 需要安装 chromadb |
| **数据量** | 适合小数据 | 支持大数据 |

**选择建议**：
- 写代码测试、临时用 → InMemoryVectorStore
- 做项目、数据要保存 → Chroma 或其他持久化向量库

### 3.3 相似度搜索的原理

**相似度搜索**就是找和查询向量最像的向量：

```
查询向量：[0.1, 0.3, 0.5, ...]

向量库中的向量：
文档1向量：[0.12, 0.28, 0.51, ...]  ← 最像，排第一
文档2向量：[0.2, 0.5, 0.1, ...]
文档3向量：[0.8, 0.1, 0.2, ...]
```

**相似度计算**：通常用余弦相似度（第8篇学过），值越接近 1 越相似。

### 3.4 Collection（集合）是什么？

Chroma 中的 `collection_name` 就像数据库中的"表"：

```
Chroma 数据库
├── collection "test"
│   ├── 文档1 + 向量1
│   ├── 文档2 + 向量2
│   └── ...
├── collection "products"
│   ├── 商品1 + 向量1
│   └── ...
└── collection "articles"
    └── ...
```

不同集合之间的数据是隔离的，可以存不同类型的数据。

## 🎯 四、常见使用模式

### 4.1 基本使用流程

```python
from langchain_core.vectorstores import InMemoryVectorStore
from langchain_community.embeddings import DashScopeEmbeddings

# 1. 创建向量存储
vector_store = InMemoryVectorStore(embedding=DashScopeEmbeddings())

# 2. 添加文档
vector_store.add_documents(documents)

# 3. 搜索
results = vector_store.similarity_search("你的问题", k=3)

# 4. 查看结果
for doc in results:
    print(doc.page_content)
```

### 4.2 Chroma 持久化使用

```python
from langchain_chroma import Chroma
from langchain_community.embeddings import DashScopeEmbeddings

# 1. 创建（第一次会自动创建目录）
vector_store = Chroma(
    collection_name="my_knowledge",
    embedding_function=DashScopeEmbeddings(),
    persist_directory="./chroma_db"
)

# 2. 添加数据
vector_store.add_documents(documents)

# 3. 下次程序启动时，直接加载已有数据
vector_store = Chroma(
    collection_name="my_knowledge",
    embedding_function=DashScopeEmbeddings(),
    persist_directory="./chroma_db"
)
# 数据还在，可以直接搜索
results = vector_store.similarity_search("问题")
```

> 💡 Chroma 会自动持久化，不需要手动调用 save 之类的方法。

### 4.3 带分数的相似度搜索

```python
# 返回相似度分数（0到1之间，越大越相似）
results = vector_store.similarity_search_with_score(
    "问题",
    k=3
)

for doc, score in results:
    print(f"相似度：{score:.4f}")
    print(f"内容：{doc.page_content[:50]}...")
```

### 4.4 按 ID 获取文档

```python
# 根据 ID 获取指定文档
doc = vector_store.get("id1")
print(doc)
```

### 4.5 更新文档

```python
# 先删除旧的，再添加新的
vector_store.delete(["id1"])
vector_store.add_documents([new_doc], ids=["id1"])
```

## ⚠️ 五、注意事项

1. **Embedding 模型要一致**：存数据和搜数据必须用同一个 Embedding 模型，否则向量空间不一样，搜不准
2. **参数名不一样**：Chroma 用 `embedding_function`，InMemoryVectorStore 用 `embedding`，注意区分
3. **Chroma 需要安装依赖**：使用前要确保安装了 `langchain-chroma` 和 `chromadb`
4. **ID 要唯一**：手动指定 ID 时，要保证每个 ID 唯一，重复会覆盖
5. **k 值要合理**：搜索时 k 不要太大也不要太小，一般 3-5 个比较合适
6. **向量维度要一致**：同一个集合里的向量维度必须相同（由 Embedding 模型决定）

## 📝 六、其他向量数据库

LangChain 支持很多种向量数据库：

| 向量数据库 | 特点 | 适用场景 |
|-----------|------|---------|
| `InMemoryVectorStore` | 内存存储，无需安装 | 开发测试 |
| `Chroma` | 轻量、本地、易上手 | 中小项目、本地部署 |
| `FAISS` | Facebook 开源，速度快 | 高性能检索 |
| `Milvus` | 开源、分布式、功能强 | 大规模生产环境 |
| `Pinecone` | 云服务、托管式 | 不想自己运维 |
| `Weaviate` | 开源、功能丰富 | 复杂场景 |
| `Qdrant` | Rust 编写，高性能 | 性能要求高 |
| `PGVector` | PostgreSQL 扩展 | 已有 PG 数据库 |

> 💡 入门推荐 Chroma，简单易用，本地就能跑。

## 🔍 七、完整 RAG 流程

向量存储是 RAG（检索增强生成）的核心组件，完整流程：

```python
from langchain_community.document_loaders import TextLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_chroma import Chroma
from langchain_community.embeddings import DashScopeEmbeddings
from langchain_community.chat_models.tongyi import ChatTongyi
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough

# ========== 第一步：准备数据 ==========
# 1. 加载文档
loader = TextLoader("knowledge.txt", encoding="utf-8")
docs = loader.load()

# 2. 分割文本
splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50
)
split_docs = splitter.split_documents(docs)

# 3. 存入向量库
embeddings = DashScopeEmbeddings()
vector_store = Chroma(
    collection_name="knowledge",
    embedding_function=embeddings,
    persist_directory="./chroma_db"
)
vector_store.add_documents(split_docs)

# ========== 第二步：检索问答 ==========
# 4. 创建检索器
retriever = vector_store.as_retriever(search_kwargs={"k": 3})

# 5. 构建提示词模板
prompt = ChatPromptTemplate.from_template("""
请根据以下上下文回答问题。
如果上下文没有答案，请回答"我不知道"。

上下文：{context}

问题：{question}

回答：
""")

# 6. 构建 RAG 链
model = ChatTongyi(model="qwen-plus")

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | model
)

# 7. 提问
question = "什么是向量数据库？"
answer = rag_chain.invoke(question)
print(f"问：{question}")
print(f"答：{answer.content}")
```

## 💡 八、完整示例模板

```python
from langchain_core.vectorstores import InMemoryVectorStore
from langchain_chroma import Chroma
from langchain_community.embeddings import DashScopeEmbeddings
from langchain_core.documents import Document

# 准备测试数据
documents = [
    Document(page_content="Python 是一种简单易学的编程语言", metadata={"source": "python"}),
    Document(page_content="Java 是一种面向对象的编程语言", metadata={"source": "java"}),
    Document(page_content="LangChain 是一个大模型应用开发框架", metadata={"source": "langchain"}),
]

# ========== 示例 1：内存向量存储 ==========
print("=== 内存向量存储 ===")

# 创建
vector_store = InMemoryVectorStore(embedding=DashScopeEmbeddings())

# 添加
vector_store.add_documents(
    documents,
    ids=["id1", "id2", "id3"]
)
print("添加了 3 个文档")

# 搜索
results = vector_store.similarity_search("什么编程语言简单？", k=2)
print(f"\n搜索结果（前2个）：")
for i, doc in enumerate(results):
    print(f"  {i+1}. {doc.page_content}")

# 带分数搜索
results_with_score = vector_store.similarity_search_with_score("Python", k=2)
print(f"\n带分数搜索：")
for doc, score in results_with_score:
    print(f"  分数: {score:.4f} - {doc.page_content}")

# 删除
vector_store.delete(["id3"])
print(f"\n删除后剩余：{len(vector_store.get()['documents'])} 个文档")

# ========== 示例 2：Chroma 持久化存储 ==========
print("\n=== Chroma 持久化存储 ===")

# 创建
chroma_store = Chroma(
    collection_name="test_collection",
    embedding_function=DashScopeEmbeddings(),
    persist_directory="./data/chroma_db"
)

# 添加
chroma_store.add_documents(documents, ids=["id1", "id2", "id3"])
print("添加了 3 个文档到 Chroma")

# 搜索
results = chroma_store.similarity_search("大模型框架", k=1)
print(f"搜索结果：{results[0].page_content}")

# 验证持久化：重新加载
print("\n重新加载 Chroma（验证持久化）：")
chroma_store2 = Chroma(
    collection_name="test_collection",
    embedding_function=DashScopeEmbeddings(),
    persist_directory="./data/chroma_db"
)
results = chroma_store2.similarity_search("编程语言", k=2)
print(f"重新加载后搜索到 {len(results)} 个结果")
for doc in results:
    print(f"  - {doc.page_content}")
```

## 📊 九、向量存储操作总结

| 操作 | 方法 | 说明 |
|-----|------|------|
| **创建** | `Chroma(...)` / `InMemoryVectorStore(...)` | 创建向量存储实例 |
| **添加** | `add_documents(docs, ids=...)` | 添加文档和向量 |
| **删除** | `delete(ids)` | 按 ID 删除 |
| **搜索** | `similarity_search(query, k)` | 相似度搜索，返回文档 |
| **带分搜索** | `similarity_search_with_score(query, k)` | 带相似度分数 |
| **获取** | `get(ids)` | 按 ID 获取文档 |
| **检索器** | `as_retriever()` | 转成 Retriever，用于链 |

## 🔗 十、知识串联

| 前面学过的 | 和向量存储的关系 |
|-----------|----------------|
| **Document Loader** | 加载文档，作为向量存储的输入 |
