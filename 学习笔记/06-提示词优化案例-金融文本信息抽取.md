# 06 - 提示词优化案例 - 金融文本信息抽取

> 学习使用 Few-Shot 提示词从金融文本中抽取结构化信息，输出 JSON 格式

---

## 📚 一、代码概述

本代码演示了**金融文本信息抽取**任务，通过 Few-Shot 提示词让大模型从非结构化的金融新闻中提取出结构化的字段（日期、股票名称、开盘价等），并以 JSON 格式输出。

### 1.1 核心功能

- ✅ 从自然语言文本中抽取指定字段
- ✅ 输出标准 JSON 格式
- ✅ 缺失信息用"无"填充
- ✅ Few-Shot 示例引导输出格式

### 1.2 抽取字段

| 字段 | 说明 | 示例 |
|-----|------|------|
| 日期 | 新闻发布日期 | 2023-01-10 |
| 股票名称 | 涉及的股票 | 强大科技A股 |
| 开盘价 | 当日开盘价格 | 100人民币 |
| 收盘价 | 当日收盘价格 | 102人民币 |
| 成交量 | 当日成交量 | 520000 |

## 🔧 二、核心代码解析

### 2.1 定义抽取 schema

```python
schema = ['日期', '股票名称', '开盘价', '收盘价', '成交量']
```

**Schema 的作用**：
- 明确告诉模型要抽取哪些字段
- 保证输出的字段名一致
- 方便后续解析和使用

### 2.2 准备示例数据

```python
examples_data = [       # 示例数据
    {
        "content": "2023-01-10, 股市震荡。股票强大科技A股今日开盘价100人民币, 一度飙升至105人民币, 随后回落至98人民币, 最终以102人民币收盘, 成交量达到520000。",
        "answers": {
            "日期": "2023-01-10",
            "股票名称": "强大科技A股",
            "开盘价": "100人民币",
            "收盘价": "102人民币",
            "成交量": "520000"
        }
    },
    {
        "content": "2024-05-16, 股市利好。股票英伟达美股今日开盘价105美元, 一度飙升至109美元, 随后回落至100美元, 最终以116美元收盘, 成交量达到3560000。",
        "answers": {
            "日期": "2024-05-16",
            "股票名称": "英伟达美股",
            "开盘价": "105美元",
            "收盘价": "116美元",
            "成交量": "3560000"
        }
    }
]
```

**示例设计要点**：
- 输入是自然语言文本
- 输出是结构化的 JSON
- 两个示例格式完全一致
- 覆盖了不同的股票和价格

### 2.3 构建系统提示词

```python
messages = [
    {"role": "system","content": f"你帮我完成信息抽取，我给你句子，你抽取{schema}信息，按JSON字符串输出，如果某些信息不存在，用无代替"}
]
```

**系统提示词包含的要素**：

| 要素 | 内容 | 作用 |
|-----|------|------|
| 任务说明 | "你帮我完成信息抽取" | 明确任务类型 |
| 字段列表 | schema 中的字段 | 明确抽什么 |
| 输出格式 | "按JSON字符串输出" | 明确输出格式 |
| 异常处理 | "信息不存在用无代替" | 处理边界情况 |

### 2.4 加入 Few-Shot 示例

```python
for chunk in examples_data:
    messages.append({"role":"user","content":chunk['content']})
    messages.append({"role":"assistant","content":json.dumps(chunk['answers'],ensure_ascii=False)})
```

**关键点**：
- 用 `json.dumps()` 把字典转成 JSON 字符串
- `ensure_ascii=False` 保证中文正常显示
- 示例的输出格式必须严格一致

### 2.5 执行抽取

```python
for q in questions:
    response = client.chat.completions.create(
        model="qwen-plus",
        messages=messages + [{"role":"user","content":f"请根据以上例子，对以下问题进行抽取{q}"}]
    )
    print(response.choices[0].message.content)
```

## 💡 三、关键知识点

### 3.1 什么是信息抽取？

**信息抽取（Information Extraction, IE）** 是从非结构化文本中提取出结构化信息的技术。

```
非结构化文本  →  [大模型]  →  结构化数据
（自然语言）                    （JSON/表格）
```

**应用场景**：
- 从新闻中提取公司、人物、事件
- 从合同中提取金额、日期、甲方乙方
- 从简历中提取学历、工作经历、技能
- 从医疗记录中提取症状、诊断、用药

### 3.2 为什么用 JSON 输出？

| 优点 | 说明 |
|-----|------|
| **结构化** | 字段清晰，便于程序解析 |
| **标准化** | 格式统一，方便后续处理 |
| **可验证** | 可以校验字段是否完整 |
| **易存储** | 可以直接存入数据库 |

> 💡 大模型输出的是文本，但如果能让它输出 JSON，就能直接被程序使用，这是连接 AI 和业务系统的关键。

### 3.3 Few-Shot 在信息抽取中的作用

1. **明确输出格式**：模型看到示例就知道要输出 JSON
2. **统一字段名称**：保证每次输出的字段名一样
3. **展示抽取规则**：通过示例隐含地告诉模型怎么抽
4. **处理缺失值**：示例展示了完整情况，配合系统提示处理缺失

## 🎯 四、输出格式控制技巧

### 4.1 确保输出纯 JSON

有时候模型会输出多余的解释，比如：
```
好的，这是抽取的结果：
{
  "日期": "2025-06-16",
  ...
}
```

**优化方法**：
- 在系统提示词中强调"只输出JSON，不要其他内容"
- 示例中只放纯 JSON，不要其他文字
- 可以用"```json"标记代码块

### 4.2 处理缺失字段

系统提示词中明确："如果某些信息不存在，用无代替"

这样可以避免：
- 模型自己编造信息（幻觉）
- 输出的 JSON 字段不完整
- 后续解析报错

### 4.3 字段值的标准化

可以在提示词中指定格式要求：
- 日期统一为 YYYY-MM-DD 格式
- 金额保留两位小数
- 百分比用数字表示（如 0.85 还是 85%）

---

## ⚠️ 五、注意事项

1. **字段名要准确**：示例中的字段名和系统提示词要完全一致
2. **示例要典型**：覆盖常见的表述方式和边界情况
3. **输出格式严格**：所有示例的输出格式必须完全相同
4. **处理中文编码**：使用 `ensure_ascii=False` 避免中文变成 Unicode 编码
5. **验证输出**：实际使用时最好加一层 JSON 解析校验，失败则重试
6. **防止幻觉**：明确要求"信息不存在用无代替"，不要编造

## 📝 六、进阶应用

### 6.1 嵌套 JSON 抽取

可以抽取更复杂的嵌套结构：

```json
{
  "公司信息": {
    "名称": "ABC公司",
    "股票代码": "000001"
  },
  "财务数据": {
    "营收": "100亿",
    "利润": "10亿"
  }
}
```

### 6.2 批量抽取

从一篇长文中抽取多个实体：

```json
{
  "人物": ["张三", "李四"],
  "公司": ["ABC公司", "XYZ公司"],
  "事件": ["并购", "发布新品"]
}
```

### 6.3 关系抽取

不仅抽取实体，还抽取实体之间的关系：

```json
{
  "实体对": [
    {"主体": "张三", "关系": "任职于", "客体": "ABC公司"}
  ]
}
```

## 🔍 七、与传统方法的对比

| 方法 | 优点 | 缺点 | 适用场景 |
|-----|------|------|---------|
| **正则表达式** | 精确、快 | 只能抽固定模式 | 格式非常固定的文本 |
| **规则引擎** | 可控 | 规则多了难维护 | 领域明确、规则清晰 |
| **NER 模型** | 准确率高 | 需要标注数据训练 | 有大量标注数据 |
| **大模型 Few-Shot** | 灵活、无需训练 | 可能出错、成本高 | 快速原型、小批量 |

> 💡 大模型的优势是"零样本/少样本就能干活"，特别适合快速验证和数据量不大的场景。
