## 📚 一、代码概述

本代码演示了**金融文本分类**的提示词工程实践，通过给模型提供几个示例（Few-Shot），让模型学会将金融文本自动分类到指定类别。

### 1.1 核心功能

- ✅ 使用 Few-Shot 提示词技术
- ✅ 将金融文本分为 4 个类别
- ✅ 支持"不清楚"类别兜底

### 1.2 分类类别

| 类别 | 说明 |
|-----|------|
| 新闻报道 | 宏观经济、市场动态等新闻 |
| 财务报告 | 公司财务数据、报表等 |
| 公司公告 | 公司发布的正式公告 |
| 分析师报告 | 行业分析、投资建议等 |

### 2.3 构建提示词

```python
messages =[
    {"role": "system","content": "你是金融专家，将文本分类为［'新闻报道'，'财务报道'，'公司公告'，'分析师报告'］,不清楚的分类为不清楚类别"}
]

# 把示例数据加入对话历史
for key,value in examples_data.items():
    messages.append({"role":"user","content":value})
    messages.append({"role":"assistant","content":key})
```

**构建后的 messages 结构：**

```
[
  system: 你是金融专家，将文本分类为...
  user: 今日，股市经历了一轮震荡...       ← 示例1输入
  assistant: 新闻报道                      ← 示例1输出
  user: 本公司年度财务报告显示...          ← 示例2输入
  assistant: 财务报告                      ← 示例2输出
  user: 本公司高兴地宣布...                ← 示例3输入
  assistant: 公司公告                      ← 示例3输出
  user: 最新的行业分析报告指出...          ← 示例4输入
  assistant: 分析师报告                    ← 示例4输出
]
```

### 2.4 逐个分类

```python
for q in questions:
    response = client.chat.completions.create(
        model="qwen-plus",
        messages=messages + [{"role": "user", "content": f"请给出分类{q}"}]
    )
    print(response.choices[0].message.content)
```

**关键技巧**：每次调用都带上完整的示例 + 当前问题，模型就会模仿示例的格式给出分类结果。

## 💡 三、关键知识点

### 3.1 什么是 Few-Shot 学习？

| 术语 | 含义 |
|-----|------|
| **Zero-Shot** | 不给例子，直接让模型做任务 |
| **One-Shot** | 给 1 个例子 |
| **Few-Shot** | 给几个例子（通常 2-10 个） |
| **Many-Shot** | 给很多例子（类似微调） |

**效果对比：**
```
Zero-Shot  →  效果一般，模型可能理解偏差
Few-Shot   →  效果大幅提升，模型能准确理解任务要求
Fine-tune  →  效果最好，但成本高、需要数据
```

### 3.2 Few-Shot 的工作原理

大模型在预训练阶段学到了大量的"模式识别"能力。当你给它几个例子时，它能：

1. **识别模式**：哦，这是一个分类任务
2. **理解规则**：原来这类文本叫"新闻报道"
3. **模仿输出**：我也按这个格式回答

> 💡 就像老师给你几道例题，你学会了解题方法，然后做新题。

### 3.3 示例设计的技巧

| 技巧 | 说明 |
|-----|------|
| **示例要典型** | 选最能代表该类别的文本 |
| **类别要均衡** | 每个类别给差不多数量的示例 |
| **覆盖边界情况** | 给一些容易混淆的例子 |
| **输出格式统一** | 所有示例的输出格式要一致 |
| **数量适中** | 2-5 个通常就够了，太多会浪费 token |

## 🎯 四、提示词优化思路

### 4.1 原始提示词（可能效果不好）

```
请把这段文本分类：{text}
```

**问题**：太模糊，模型不知道有哪些类别，也不知道输出格式。

### 4.2 优化后（加类别说明）

```
请将以下文本分类为：新闻报道、财务报告、公司公告、分析师报告。
文本：{text}
分类结果：
```

**改进**：明确了分类选项，但模型可能还是理解不准确。

### 4.3 再优化（Few-Shot）

```
请将文本分类，参考示例：

文本：今日，股市经历了一轮震荡...
分类：新闻报道

文本：本公司年度财务报告显示...
分类：财务报告

...

文本：{text}
分类：
```

**效果**：模型准确率大幅提升！

## ⚠️ 五、注意事项

1. **示例顺序可能影响结果**：可以试试不同的排列顺序
2. **不要给太多示例**：够用就行，太多会增加 token 消耗
3. **示例要多样化**：每个类别的示例要有代表性
4. **处理边界情况**：一定要考虑"不属于任何类别"的情况，设置兜底类别
5. **输出格式要严格**：示例的输出格式要完全一致，模型才会模仿

## 🔍 七、进阶：提示词工程的一般步骤

1. **明确任务**：清楚地定义要做什么
2. **设定角色**：给模型一个专业身份（如"你是金融专家"）
3. **说明规则**：列出分类标准、输出格式等
4. **给出示例**：2-5 个典型示例（Few-Shot）
5. **测试调优**：用测试数据验证效果，不断调整
6. **处理边界**：考虑异常情况、兜底方案
