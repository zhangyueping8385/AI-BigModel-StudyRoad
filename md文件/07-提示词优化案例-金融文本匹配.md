# 07 - 提示词优化案例 - 金融文本匹配

> 学习使用 Few-Shot 提示词判断两段文本是否语义匹配，是/否二分类任务

---

## 📚 一、代码概述

本代码演示了**文本匹配（Text Matching）**任务，通过 Few-Shot 提示词让大模型判断两段金融文本是否表达相同或相似的意思，输出"是"或"不是"。

### 1.1 核心功能

- ✅ 判断两段文本是否语义匹配
- ✅ 二分类输出（是/不是）
- ✅ Few-Shot 示例引导判断标准

### 1.2 应用场景

- 新闻去重：判断两条新闻是不是同一件事
- 问答匹配：判断用户问题和知识库问题是否匹配
- 语义相似度：计算两个句子的相似程度
- 文本聚类：把相似的文本归为一类

## 🔧 二、核心代码解析

### 2.1 准备示例数据

```python
examples_data = {
    "是": [
        ("公司ABC发布了季度财报，显示盈利增长。", "财报披露，公司ABC利润上升。"),
        ("公司ITCAST发布了年度财报，显示盈利大幅度增长。", "财报披露，公司ITCAST更赚钱了。")
    ],
    "不是": [
        ("黄金价格下跌，投资者抛售。", "外汇市场交易额创下新高。"),
        ("央行降息，刺激经济增长。", "新能源技术的创新。")
    ]
}
```

**示例设计要点**：

| 类别 | 示例特点 |
|-----|---------|
| **是（匹配）** | 表述不同但意思相同（财报盈利增长 = 利润上升） |
| **不是（不匹配）** | 完全不相关的两个话题 |

> 💡 正例和负例都要有，模型才能学会判断边界。

### 2.2 准备测试数据

```python
questions = [
    ("利率上升，影响房地产市场。", "高利率对房地产有一定的冲击。"),
    ("油价大幅度下跌，能源公司面临挑战。", "未来智能城市的建设趋势越加明显。"),
    ("股票市场今日大涨，投资者乐观。", "持续上涨的市场让投资者感到满意。")
]
```

### 2.3 构建系统提示词

```python
messages = [
    {"role": "system", "content": f"你帮我完成文本匹配，我给你2个句子，被[]包围，你判断它们是否匹配，回答是或不是，请参考如下示例： "},
]
```

**系统提示词要素**：
- 任务：文本匹配
- 输入：2个句子
- 输出：是或不是
- 方式：参考示例

### 2.4 加入 Few-Shot 示例

```python
for key,value in examples_data.items():
    messages.append({"role":"user","content":f"{value}"})
    messages.append({"role":"assistant","content":f"{key}"})
```

**构建后的对话结构**：

```
system: 你帮我完成文本匹配...
user: [("公司ABC发布了季度财报...", "财报披露..."), ("公司ITCAST发布了...", "财报披露...")]
assistant: 是
user: [("黄金价格下跌...", "外汇市场交易额..."), ("央行降息...", "新能源技术...")]
assistant: 不是
```

> ⚠️ 注意：这里的示例格式稍微有点特殊——一次给了两对句子，然后给出统一答案。更标准的做法是一对句子对应一个答案。

### 2.5 执行匹配

```python
for q in questions:
    response = client.chat.completions.create(
        model="qwen-plus",
        messages=messages + [{ "role":"user","content":f"请你根据以上例子对以下问题进行匹配{q}" }]
    )
    print(response.choices[0].message.content)
```

## 💡 三、关键知识点

### 3.1 什么是文本匹配？

**文本匹配（Text Matching）** 是判断两段文本在语义上是否相同或相似的任务。

```
句子A: "公司盈利增长了"
句子B: "公司利润上升了"
    ↓
[文本匹配模型]
    ↓
  输出：是（语义匹配）
```

### 3.2 文本匹配的难度层级

| 难度 | 示例 | 说明 |
|-----|------|------|
| **简单** | "今天天气好" vs "今天天气不错" | 同义词替换 |
| **中等** | "公司盈利增长" vs "财报显示利润上升" | 句式不同，意思相同 |
| **困难** | "央行降息" vs "房贷利率下降" | 需要推理和背景知识 |
| **歧义** | "我喜欢苹果" vs "我喜欢苹果手机" | 多义词，需要上下文 |

### 3.3 Few-Shot 在文本匹配中的作用

1. **明确判断标准**：通过示例告诉模型"什么样算匹配"
2. **统一输出格式**：保证输出是"是"或"不是"
3. **展示边界情况**：正例和负例对比，让模型学会区分
4. **调整严格程度**：通过示例控制匹配的宽松/严格

## 🎯 四、文本匹配的应用场景

### 4.1 问答系统（FAQ）

用户问："怎么退款？"
知识库中有："退货流程是什么？"
→ 判断匹配 → 返回退货流程的答案

### 4.2 新闻去重

多个媒体报道同一件事
→ 判断语义重复 → 只保留一篇

### 4.3 抄袭检测

学生作文 vs 网上范文
→ 判断相似度 → 检测抄袭

### 4.4 对话系统

用户说："我要查快递"
系统识别：这是"物流查询"意图
→ 匹配到对应的处理流程

## ⚠️ 五、注意事项

### 5.1 示例设计原则

1. **正负例均衡**：匹配和不匹配的示例数量差不多
2. **覆盖各种情况**：简单匹配、复杂匹配、容易混淆的都要有
3. **格式一致**：所有示例的输入输出格式要统一
4. **难度递进**：可以从简单到复杂排列

### 5.2 常见问题

**Q: 模型总是回答"是"怎么办？**
> 增加负例的数量，或者让负例更接近正例（更难区分），迫使模型学习真正的判断标准。

**Q: 输出不是"是"或"不是"，而是一大段解释？**
> 在系统提示词中强调"只回答是或不是，不要解释"，并确保示例中只有"是"或"不是"两个字。

**Q: 匹配的标准太宽松/太严格了？**
> 通过调整示例来控制。想要严格，就给一些"看起来像但其实不匹配"的负例；想要宽松，就给一些"表述差异大但意思相同"的正例。

## 📝 六、进阶：从二分类到相似度评分

### 6.1 输出相似度分数

可以让模型输出 0-1 的相似度分数，而不只是"是/不是"：

```
句子A: 公司盈利增长
句子B: 财报显示利润上升
相似度: 0.9
```

### 6.2 多维度匹配

可以从多个维度判断匹配度：

```json
{
  "整体匹配度": 0.85,
  "主题匹配": 0.9,
  "情感匹配": 0.7,
  "实体匹配": 0.95
}
```

## 🔍 七、与传统方法的对比

| 方法 | 原理 | 优点 | 缺点 |
|-----|------|------|------|
| **关键词匹配** | 看共同关键词数量 | 快、简单 | 不理解语义 |
| **余弦相似度** | 计算向量夹角 | 比关键词好 | 还是不够智能 |
| **Sentence-BERT** | 专用句向量模型 | 准确率高 | 需要训练/微调 |
| **大模型 Few-Shot** | 让大模型判断 | 灵活、理解语义 | 慢、成本高 |

> 💡 大模型的优势是能理解深层语义，甚至能做推理判断。但对于简单的匹配任务，传统方法可能更高效。

## 💡 八、提示词优化建议

### 优化前（简单版）

```
判断下面两个句子是否匹配：
句子1：{sent1}
句子2：{sent2}
回答是或不是。
```

### 优化后（Few-Shot 版）

```
你是一个文本匹配专家，判断两个句子是否表达相同的核心意思。
注意：
- 表述不同但意思相同 → 是
- 话题不同 → 不是
- 同一事件的不同报道 → 是

示例1：
句子1：公司ABC发布了季度财报，显示盈利增长。
句子2：财报披露，公司ABC利润上升。
答案：是

示例2：
句子1：黄金价格下跌，投资者抛售。
句子2：外汇市场交易额创下新高。
答案：不是

现在请判断：
句子1：{sent1}
句子2：{sent2}
答案：
```
